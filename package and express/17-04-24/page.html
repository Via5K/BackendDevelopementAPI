Package.json

Package.json it contains all the npm modules we are using.
we can create that using npm init.
Basic information like name and version is stored and required in pacakge.json file.

When we give the version as 1.1.0
Here those 3 means-  Major.minor.patches
Where Major means it is totally new like it can break or bring functionality which was not there.
Minor means backwards compatible, and fix of previous version.
Patches means that it is kind of a bug fix.


Using ~ in front of version means that you are allowing npm to install the updated patches, but making sure that npm install it from Minor version only.
i.e 1.1.* star meaning the latest patch.

When using ^ in fron to fversion that means you are allowing npm to install the updated minors as well. But making sure that Mjor reamains same.
i.e 1.*.* star meaning the latest version available of minor and patches.
